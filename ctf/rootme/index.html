What is the hidden directory?

Existem 2 diretórios interessantes aqui neste resultado:
- /panel/ que é de fato um painel "escondido" porque não existia referência para ele da página principal.
- /uploads/ onde provavelmente estarão possíveis arquivos que conseguirmos enviar para o servidor.

Acessando o diretório /panel/ me deparo com uma página de upload de arquivos.

Getting a Shell

Find a form to upload and get a reverse shell, and find the flag.

A primeira coisa que tentei foi fazer o upload de um arquivo de reverse shell em PHP (rev.php) para ser executado no servidor.

Referência github.com/pentestmonkey/php-reverse-shell/blob/master/php-reverse-shell.php

Porém aqui me deparei com um erro dizendo que a extensão .php não é permitida.

Para fazer esse bypass, é possível alterar a extensão do arquivo para .php5

Referência book.hacktricks.xyz/pentesting-web/file-upload

$ mv rev.php rev.php5

Agora faço o upload novamente, porém com sucesso.

Preciso agora executar esse arquivo PHP que foi feito o upload para o servidor.
É possível fazer isso por meio do diretório /uploads/ que encontramos anteriormente.

Antes de executar o reverse shell, precisamos deixar o IP:PORT configurados no arquivo escutando a conexão reversa:

$ nc -lvnp 1337 Listening on 0.0.0.0 1337

Com o netcat rodando, eu uso o cURL para fazer um resição que vai executar o nosso payload no servidor alvo:

$ curl http://10.10.149.144/uploads/rev.php5

Ao ser executado, o netcat recebe então a conexão do shell reverso interativo vinda do servidor alvo:
